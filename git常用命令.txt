git status 状态查询
git status -s

git commit -m 提交到git仓库 m后面是描述

git commit -a -m git就会自动把所有已经跟踪过的文件暂存起来一并提交 从而跳过 git add步骤

git add 文件名  添加到暂存区 
git add . 将工作区的文件都提交到暂存区 

git checkout -- index.html 撤销修改 还原成Git仓库中所保留的版本
操作的结果：所有的修改都会丢失且无法恢复！危险性比较高，请慎重操作


git reset HEAD 文件名  取消已暂存的文件  从暂存区移除相对应的文件

从Git仓库和工作区中同时移除
git rm -f 文件名

git rm --cached 文件名
